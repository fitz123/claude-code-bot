## Reference: Cron fields not documented in README

These fields exist in code but are not mentioned in README's "Add a Cron" section:
- `type: "script" | "llm"` — script-mode crons run shell commands instead of LLM prompts
- `timeout` — per-cron timeout in milliseconds
- `maxBudget` — cost cap per run in dollars
- `deliveryThreadId` — Telegram forum topic ID for delivery
- `enabled` — skip plist generation when false (default: true)
- `model` — per-cron model override
