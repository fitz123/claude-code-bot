## Reference: Feature inventory (from production use)

Sources: `reference/tasks/bot-comparison/competitor-analysis.md` and `reference/tasks/bot-ors/launch-draft.md` in workspace.

**Bot core:**
- Multi-platform (Telegram + Discord) via PlatformContext abstraction
- Multi-agent: N agents with separate workspaces, models, system prompts, fallback models
- Session management: LRU eviction, crash circuit breaker (5 crashes â†’ block), exponential backoff, persistent resume across restarts
- Message queue: 3s debounce + 20-message mid-turn collect buffer
- Mid-turn message injection via PreToolUse hook (file-based IPC with ack protocol)
- Streaming updates: progressive message edits with configurable per-binding toggle
- Voice transcription: local whisper-cli (free, offline)
- Document upload: PDF, TXT, CSV, JSON, XML, HTML, ZIP, GZ (max 20MB)
- File output via outbox directory
- Emoji reactions forwarded to Claude as context (JSONL audit log)
- Forum topics: per-topic agent routing with isolated sessions
- Polling watchdog with getMe heartbeat
- Graceful shutdown (SIGTERM â†’ wait busy sessions â†’ save state)
- Startup retry with exponential backoff on 409 Conflict
- Stale message protection (maxMessageAgeMs)
- @mention filtering (configurable per-binding)
- Source context injection (chat/topic/sender metadata)
- NO_REPLY suppression (agent signals "no response needed")
- Fallback model per agent
- Max turns safety limit per agent
- Configurable reasoning effort per agent

**Cron system:**
- YAML-based config â†’ launchd plists generation
- Per-cron: delivery target, timeout, budget cap, enabled toggle
- Script-mode crons (shell commands, not just LLM prompts)
- Config-level default delivery target

**Monitoring:**
- 11 Prometheus metrics (tokens, cost, turn duration, sessions, crashes, API errors)
- prom-client HTTP server

**Workspace template:**
- Platform hooks (protect-files, guardian, auto-stage, session-end-commit, session-start-recovery, inject-message)
- Platform rules (safety, bot-operations, ADR governance, output formatting, show-files)
- Skills framework (workspace-health, memory-consolidation)
- Memory system (auto files with frontmatter, MEMORY.md index)
